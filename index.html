Typography

iGraal uses two typefaces: New Spirit (serif) for display and headlines, and Inter (sans-serif) for UI and body. This dual-font system creates personality through New Spirit's warmth while maintaining readability through Inter's precision.

New Spirit
Serif · Display & Headlines
Emotional, warm, personality. Used for page-level headings (H1–H3) and hero display text.
Shown with Georgia fallback (licensed font)
Inter
Sans-serif · Body & UI
Functional, clean, precise. Used for body copy, buttons, labels, captions, and all interactive elements.
Google Fonts — Variable weight 100–900

✅ Do

  • Use New Spirit only for H1–H3 level headings
  • Use Inter for all interactive elements (buttons, links, inputs)
  • Always use a style from the defined type ramp
  • Match each style's designated weight exactly

❌ Don't

  • Use New Spirit for body copy or UI elements
  • Mix weights randomly — each style has a specific weight
  • Use custom font sizes outside the type ramp
  • Apply bold to Body styles (they are always weight 400)

Responsive Scaling

iGraal's type ramp is fixed — the same sizes are used across all breakpoints. Layout adapts through container width changes, not type scaling. This is a simpler approach than responsive type systems like eBay's.

Context Mobile (<768px) Desktop (≥768px) Behaviour
Page title (Display) 32px / 40px 32px / 40px Fixed
Section title (Headline L) 28px / 36px 28px / 36px Fixed
Card title (Headline M) 22px / 32px 22px / 32px Fixed
Body 14px / 20px 14px / 20px Fixed
Container width 100% – 32px max-width: 1200px Responsive
💡 Note: No responsive scaling tokens were found in iGraal's CSS variables. The type system remains consistent across breakpoints. Readability on mobile is maintained through line-height ratios and container padding, not by scaling font sizes.

Card System

iGraal's card system (Nalu design system) organizes UI into 3 main component families + layout containers. Cards are the primary content vehicles — every offer, promotion, and content block is delivered through this system.

🏗️ Design System

Internal system called Nalu. All card components are prefixed with their family name for consistency.

📐 4 Families

Vertical Base Card, Horizontal Base Card, Vertical Lite Card, and Island — plus supporting components.

🎯 5 Functional Groups

Commerce, Content, Promotional, Navigation, and Layout — organized by purpose, not appearance.

🧩 Composable

Cards combine with Tags, Badges, Chips, and Buttons from shared component libraries for flexible UI assembly.

Primary cashback offer cards. Used in carousels for featured and best offers — the most visible card type on the homepage and category pages.

When to use: Showcasing time-limited cashback promotions with strong visual brand presence. Featured in horizontal carousels with pagination dots.

Island Anatomy Patterns

Testimonial

Avatar image → author name → review title → review body. Used in horizontal scroll carousel (3 visible).

Promotional

Headline → description → app store links or browser extension CTA. Full-width, two-column layout inside.

FAQ

Heading → accordion items (expand/collapse). Uses Nalu accordion component.

Footer

Logo → navigation links → social icons → legal text. Multi-column layout.

✅ Do
  • Use for full-width content blocks that aren't offer cards
  • Match the island variant to content type (testimonial, CTA, FAQ)
  • Use pale orange (#FFF3EB) for promotional/brand sections
  • Use light grey (#F2F2F2) for neutral/footer content
  • Keep 32px radius for large islands, 16px for medium
❌ Don't
  • Don't use Islands for commerce offer cards
  • Don't nest Islands inside other Islands
  • Don't mix Island backgrounds arbitrarily — follow the variant system
  • Don't reduce padding below spec — generous whitespace is intentional
  • Don't add shadows to Islands — they rely on background color for definition

Card System Grouping

From a design system perspective, all cards and components group into 5 functional categories based on their purpose — not their appearance.

🛒 Commerce Cards

Drive conversion
  • Vertical Base Card → featured offers in carousels
  • Horizontal Base Card → detailed offers on shop pages
  • Vertical Base Card Lite → gift card browsing

📖 Content Cards

Inform & engage
  • Testimonial Island → social proof
  • FAQ Island → self-service support

📣 Promotional Cards

Acquire & retain
  • Promotional Island → app download, extension install CTAs
  • Stats Section → trust metrics (200+ partners, 6% avg cashback, 20 years)

🧭 Navigation Cards

Browse & discover
  • Shop Grid Button → category-based store discovery
  • Brand Logo Link → popular brands grid
  • Category Chip → filtering mechanism

📦 Layout Containers

Structural
  • Section (nalu section) → page-level content grouping
  • Island (nalu island) → styled content block within section
  • Carousel (nalu carousel) → horizontal scrolling with nav arrows + dots

Do / Don't Guidelines

✅ Do

  • Use "Ty/Twoje" (you/your) — always address the user directly
  • Use "za darmo" and "darmowy" when applicable — free is a key motivator
  • Include numbers and metrics — specificity builds trust
  • Use first-person for CTAs when appropriate ("Zapisuję się")
  • Keep headlines benefit-first, not feature-first

❌ Don't

  • Use technical/financial jargon without explanation
  • Use passive voice — always active and direct
  • Use vague promises — always quantify (%, zł, timeframes)
  • Use fear-based urgency — use positive framing instead
  • Mix formal and informal register in the same context

iGraal's motion design is subtle and functional — animations serve usability, not decoration. The system follows Material Design-inspired transition tokens with custom Nalu timing.

Motion Guidelines

✅ When to Animate

  • ✅ State changes — hover, focus, active, disabled
  • ✅ Content reveal — accordion, expandable text, carousel
  • ✅ Loading states — spinners, skeletons, progress
  • ✅ Micro-feedback — button press, checkbox, toggle

❌ Never Animate

  • ❌ Decoration alone — no parallax on scroll
  • ❌ No floating elements without purpose
  • ❌ No auto-play animations > 5s without controls
  • ❌ No animation that bypasses reduced motion

♿ Accessibility

  • Respect prefers-reduced-motion: reduce
  • Loading: aria-busy="true" + role="progressbar"
  • Carousel auto-advance pauses on hover/focus
  • No animation > 5s without user control (WCAG 2.3.3)

Either make the card non-interactive (role="article") with the inner button as the only interactive element, OR remove the inner button and make the whole card the clickable target.
