Card System

iGraal's card system (Nalu design system) organizes UI into 3 main component families + layout containers. Cards are the primary content vehicles — every offer, promotion, and content block is delivered through this system.

🏗️ Design System

Internal system called Nalu. All card components are prefixed with their family name for consistency.

📐 4 Families

Vertical Base Card, Horizontal Base Card, Vertical Lite Card, and Island — plus supporting components.

🎯 5 Functional Groups

Commerce, Content, Promotional, Navigation, and Layout — organized by purpose, not appearance.

🧩 Composable

Cards combine with Tags, Badges, Chips, and Buttons from shared component libraries for flexible UI assembly.

Island Anatomy Patterns

Testimonial

Avatar image → author name → review title → review body. Used in horizontal scroll carousel (3 visible).

Promotional

Headline → description → app store links or browser extension CTA. Full-width, two-column layout inside.

FAQ

Heading → accordion items (expand/collapse). Uses Nalu accordion component.

Footer

Logo → navigation links → social icons → legal text. Multi-column layout.

✅ Do
  • Use for full-width content blocks that aren't offer cards
  • Match the island variant to content type (testimonial, CTA, FAQ)
  • Use pale orange (#FFF3EB) for promotional/brand sections
  • Use light grey (#F2F2F2) for neutral/footer content
  • Keep 32px radius for large islands, 16px for medium
❌ Don't
  • Don't use Islands for commerce offer cards
  • Don't nest Islands inside other Islands
  • Don't mix Island backgrounds arbitrarily — follow the variant system
  • Don't reduce padding below spec — generous whitespace is intentional
  • Don't add shadows to Islands — they rely on background color for definition

Card System Grouping

From a design system perspective, all cards and components group into 5 functional categories based on their purpose — not their appearance.

🛒 Commerce Cards

Drive conversion
  • Vertical Base Card → featured offers in carousels
  • Horizontal Base Card → detailed offers on shop pages
  • Vertical Base Card Lite → gift card browsing

📖 Content Cards

Inform & engage
  • Testimonial Island → social proof
  • FAQ Island → self-service support

📣 Promotional Cards

Acquire & retain
  • Promotional Island → app download, extension install CTAs
  • Stats Section → trust metrics (200+ partners, 6% avg cashback, 20 years)

🧭 Navigation Cards

Browse & discover
  • Shop Grid Button → category-based store discovery
  • Brand Logo Link → popular brands grid
  • Category Chip → filtering mechanism

📦 Layout Containers

Structural
  • Section (nalu section) → page-level content grouping
  • Island (nalu island) → styled content block within section
  • Carousel (nalu carousel) → horizontal scrolling with nav arrows + dots

Critical ARIA Parent Roles Missing 7 instances expand_more
Rule

aria-required-parent — WCAG 2.1 §1.3.1 (Info and Relationships)

Issue

Navigation menuitem roles used without parent menu or menubar wrapper. Screen readers cannot understand the navigation structure.

Where

Main navigation bar — all 7 nav items

Fix

Wrap navigation items in <nav role="menubar"> or <ul role="menu">

open_in_new WCAG 1.3.1 Reference
Serious Links Without Discernible Text 3 instances expand_more
Rule

link-name — WCAG 2.1 §2.4.4, §4.1.2 (Link Purpose, Name/Role/Value)

Where
  • Logo link (home) — SVG without aria-label
  • Trustpilot link — empty anchor wrapping an image
  • Footer logo link — same issue as header
Fix

Add aria-label="Strona główna iGraal" to logo links, aria-label="Opinie na Trustpilot" to Trustpilot link.

open_in_new WCAG 2.4.4 Reference
Serious Nested Interactive Controls 5 instances expand_more
Rule

nested-interactive — WCAG 2.1 §4.1.2 (Name, Role, Value)

Where

Vertical Base Card — outer div[role="button"] contains inner <button> elements ("Aktywuj cashback"). Screen readers announce conflicting interactive roles; keyboard nav is ambiguous.

Fix

Either make the card non-interactive (role="article") with the inner button as the only interactive element, OR remove the inner button and make the whole card the clickable target.

open_in_new WCAG 4.1.2 Reference
Moderate Content Outside Landmarks 31 instances expand_more
Rule

region — Best practice (landmark navigation)

Where

Various content sections not wrapped in <main>, <section>, or landmark roles. Screen reader users cannot navigate by landmarks.
